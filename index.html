QnA

Q: What is changing?

A: 2018 年 7 月以降、新しく作成されるすべての EC2 リソースは、より長い ID フォーマットを受け取るようになります。新しいフォーマットは、新しく作成されるリソースにのみ適用され、既存のリソースが影響を受けることはありません。インスタンスとボリュームは、既にこの ID フォーマットを使用しています。2018 年 6 月末までに、お客様はより長い ID の使用をオプトインできるようになります。この期間内は、どの ID フォーマットリソースを割り当てるかを選択できます。また管理ツールとスクリプトを更新してより長いフォーマットのサポートを追加できます。方法は<a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/resource-ids.html">この</a>ドキュメントをご覧ください。

Q: Why is this necessary?

A: AWS の継続的な成長の速さを考えると、2018 年には特定のリソースの ID が枯渇し始めると考えられます。新しいリソースを中断なく長期的に作成するために、より長い ID フォーマットを導入する必要があります。Amazon EC2 のリソース ID はすべて、2018 年の 7 月により長い ID 形式に変わります。

Q: I already opted in for longer IDs last year. Why do I need to opt-in again?

A: 2016 年に、Amazon EC2 インスタンス、予約、ボリューム、スナップショットのみを、より長い ID フォーマットに移行しました。今回のオプトインは、残りのすべての EC2 リソースタイプの ID フォーマットを変更します。

Q: What will the new identifier format look like?

A: 新しい識別子のフォーマットは、現在の識別子のフォーマットのパターンに従っていますが、より長くなります。新しいフォーマットは – &lt;17 characters&gt; で、たとえば VPC については &quot;vpc-1234567890abcdef0&quot;、サブネットについては &quot;subnet-1234567890abcdef0&quot; のようになります。

Q: How does this impact me?

A: これは新しい形式を取り扱うためにシステムに変更を加える必要は一切ない、という意味で絶好の機会です。AWS リソースの管理にコンソールのみを使用しているのであれば影響はほとんどありませんが、より長い ID フォーマットを使用するよう、できる限り早く設定を更新する必要があります。API、SDK、また AWS CLI を使って AWS リソースをやり取りしているのであれば、リソース ID を検証または持続させる場合に、ご利用のソフトウェアによって ID フォーマットが推定されるかどうかに応じて、影響が発生する可能性があります。この場合、新しいフォーマットに対応するためにシステムを更新する必要が生じる可能性があります。<br /> 以下のいくつかの障害モードが考えられます。

Q: Will this affect existing resources?

A: いいえ。影響を受けるのは、より長いフォーマットをオプトインした後に作成したリソースのみです。リソースに長い ID または短い ID が割り当てられると、その後は変更できません。各 ID は一意で、再利用されることはありません。以前の ID フォーマットで作成したリソースでは、引き続き短い ID を使用します。新しい ID フォーマットで作成したリソースでは、オプトアウトした場合でも長い ID を使用します。

Q: When will this happen?

A: 2018 年 6 月までに、API と EC2 コンソールでより長い ID の使用をオプトインできるようになります。すべてのアカウントで、必要に応じてより長い ID にオプトイン、オプトアウトを試すことができます。2018 年 7 月 1 日以降、フォーマットを切り替えるオプションは利用できなくなり、新しく作成される EC2 リソースではより長い ID を受け取るようになります。2018 年以降に利用開始にするリージョンではすべて、より長い ID のみをサポートします。

Q: Why is there an opt-in period?

A: お客様には、新しいフォーマットをシステムでテストするための十分な時間をとっていただきたいと考えています。この移行期間により、システムを徐々にテストして更新する大きな柔軟性がもたらされ、また必要に応じて、新しいフォーマットのサポートを追加するための中断を最小限にとどめることができます。

Q: How do I opt in and out of receiving longer IDs?

A: 移行期間 (2018 年 6 月末まで) を通じ、API または EC2 コンソールを使用して選択することでより長い ID またはより短い ID を受け取れます。方法は <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/resource-ids.html">この</a>ドキュメントをご覧ください。

Q: What will happen if I take no action?

A: 移行期間中に新しいフォーマットをオプトインしない場合、2018 年 7 月 1 日より後には自動的により長い ID を受け取るようになります。この方法はお勧めしません。制御されたテストを実行する機会が提供される移行期間中に、新しいフォーマットのサポートを追加することが推奨されます。

Q: What if I prefer to keep receiving the shorter ID format after the end of June 2018?

A: ユーザー設定に関係なく、これは不可能です。

Q: When will the longer IDs’ final transition happen?

A: 2018 年 7 月に、新たに作成されたリソースはより長い ID を受け取り始めます。AWS CLI の <a href="https://docs.aws.amazon.com/cli/latest/reference/ec2/describe-id-format.html">describe-id-format</a> を使用して、各リージョンでの予定移行日付をチェックできます。

Q: If I opt in to longer IDs and then opt back out during the transition period, what will happen to resources that were created with longer IDs?

A: 一旦リソースに ID が割り当てられるとそれは変更できません。それでより長い ID で作成されたリソースについては、その後実行したアクションにかかわらず、より長い ID が保持されます。より長いフォーマットをオプトインしてリソースを作成し、その後オプトアウトした場合、オプトアウトした後も、長いリソース ID と短いリソース ID が混在することになります。長い ID を削除するには、個別にリソースを削除または終了する必要があります。それで、利用しているツールとオートメーションのテストが完了するまでは常に注意を払い、重要なリソースは作成しないようにします。

Q: What should I do if my systems are not working as expected before the transition period ends?

A: 移行期間中にシステムが正常に機能していない場合は、より長い形式の ID を一時的にオプトアウトしてシステムを修復できます。ただし、2018 年 6 月末より後には、アカウントはより長い形式の ID に自動的に移行されます。アカウント設定に関係なく、すべての新しいインスタンスはより長い形式の ID を受け付けます。そのため、移行期間終了前により長い形式の ID を使用してシステムをテストしておくことが重要です。早めにテストしてオプトインしておくことで、リソース ID が短いリソースに変更を施すための貴重な時間を確保して、システムに影響が発生するリスクを最小にできます。

Q: What will happen if I launch resources in multiple regions during the transition period?

A: リソースの ID 長は、リソースの使用を開始するリージョンにより異なります。リージョンがすでに長い ID を使用するように移行されている場合、そのリージョンで使用開始されたリソースの ID は長い形式のものになります。そうでない場合、短いリソース ID になります。したがって、移行期間中は、より短いリソース ID とより長いものが混在します。

Q: If AWS adds new regions during the transition period, will new regions support longer IDs?

A: はい。2018 年 7 月から後に加わる新しいリージョンではすべて、新規、既存のアカウントの両方にデフォルトでより長いフォーマットの ID が発行されます。

Q: What will be the default ID type for new accounts?

A: 2018 年 3 月 15 日以降に作成されるアカウントは、AWS GovCloud (米国) 以外の全 AWS リージョンで、デフォルトでより長い ID 形式を受け取ることになります。このため、新しいお客様の場合、より長い ID への移行は非常に簡単になります。新しいアカウントでリソースにより短い ID フォーマットを割り当てたい場合、上記のように、アカウントにより短い ID を簡単に再設定できます。このワークフローは、アカウントでより長い ID を受け取る準備ができるまで必要となります。
