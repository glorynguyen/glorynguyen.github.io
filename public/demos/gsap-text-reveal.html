// GSAP Logic
// Uses an imperative Timeline approach
const tl = gsap.timeline({ paused: true });

tl.to(chars, {
  keyframes: [
    { color: "#6b2849", duration: 0.1 },
    { color: "#ec008c", duration: 0.1 },
    { color: "#ffc2e3", duration: 0.2 },
    { color: "#ffffff", duration: 0.2 }
  ],
  stagger: 0.025,
  ease: "none"
});

area.onmouseenter = () => tl.play();
area.onmouseleave = () => tl.reverse();

// Motion.dev Logic
// Uses a functional/array-based approach
let animation;

area.onmouseenter = () => {
  animation = animate(chars, 
    { color: ["#222", "#6b2849", "#ec008c", "#ffc2e3", "#fff"] },
    { 
      delay: stagger(0.025),
      duration: 0.8,
      easing: "ease-in-out"
    }
  );
};

area.onmouseleave = () => animation?.reverse();

GSAP Analysis

  • State Management: Uses a "Playhead" logic. Reversing halfway is perfectly fluid without visual jumps.
  • Granular Timing: Can define exact seconds for each color stop (critical for the "Pink Flash" effect).
  • Scale: Highly optimized for animating 100+ characters via a single internal loop.
  • File Size: Slightly heavier (~25kb).
  • Imperative: Requires creating and managing a timeline object manually.

Motion.dev Analysis

  • Syntax: Declarative and clean; mapping colors to an array is very intuitive.
  • Hardware Acceleration: Uses WAAPI (native browser tech) for potential main-thread offloading.
  • Lightweight: Small footprint (~15kb).
  • Interruption Handling: Requires manual management of animation instances; rapid hovering can cause "fighting" states.
  • Linear Keyframes: Colors are evenly distributed by default; custom timing requires complex "offset" math.

The Verdict for "Huge Inc" Effect: GSAP Wins

Because this effect relies on interruptible hover states and non-linear color durations (a quick pink flash followed by a slow white settle), GSAP is the superior choice. It eliminates the "jumpy" behavior found in state-less libraries and provides the professional-grade control needed to match high-end design references exactly.
